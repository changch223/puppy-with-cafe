## User Scenarios & Testing *(mandatory)*

### User Story 1 - 詳細画面だけで「行くかどうか」を決められる (Priority: P1) 🎯

利用者はカフェ詳細を開くと、犬同伴可否に加えて、営業時間（と今営業中か）、犬向け設備（店内OK・テラス・大型犬・犬用メニュー）、電話・予約情報、公式サイトや Instagram へのリンク、運営が公式SNSで確認したメモ（確認日つき）が見える。アプリの外に出ずに「今から行けるか・うちの子と入れるか・どう予約するか」をおおむね判断でき、最終確認したい人はリンク先の公式情報に飛べる。

**Why this priority**: 本フィーチャーの核心。「アプリを見ればどこに行くか決められる」を実現する。

**Independent Test**: 情報が登録されたカフェの詳細を開き、営業時間＋営業中バッジ・犬設備4バッジ・電話（タップで発信）・予約情報・公式リンク集・転記メモ（確認日つき）が表示されること。未登録の項目はセクションごと非表示になり「不明」を「なし」と誤認させないこと。

**Acceptance Scenarios**:

1. **Given** 営業時間（構造化）が登録されたカフェ, **When** 現在時刻が営業時間内に詳細/一覧を見る, **Then** 「営業中」バッジが表示される（時間外なら「営業時間外」、定休曜日なら「本日定休」）。
2. **Given** 営業時間がフリーテキストのみのカフェ, **When** 詳細を見る, **Then** テキストが確認日つきで表示され、バッジは表示されない。
3. **Given** 犬向け設備が登録されたカフェ, **When** 詳細を見る, **Then** 店内OK/テラスOK/大型犬OK/犬用メニューが ✓/✕ で示され、未入力の項目は「不明」として区別される。
4. **Given** 公式リンク（HP/Instagram/X等）が登録されている, **When** リンクをタップする, **Then** 外部ブラウザ/アプリで開く。
5. **Given** 運営転記メモ（例:「公式Instagramより: テラス席は雨天クローズ」）がある, **When** 詳細を見る, **Then** メモが**出どころと確認日つき**で表示され、確定情報と誤認させない。
6. **Given** 電話番号が登録されている, **When** タップする, **Then** 電話発信の確認が出る。

### Edge Cases

- **情報未登録**: 未登録の項目・セクションは非表示（空欄・「なし」と誤認させない）。犬設備の未入力は「不明」表示。
- **営業時間の鮮度**: 営業時間・基本情報には確認日を併記し、古い場合（1年超）は既存の鮮度警告の対象に含める。
- **深夜営業・複数時間帯**: 同一曜日に複数の時間帯（例: 9:00-12:00, 13:00-18:00）を扱える。日跨ぎ営業（close < open）はv1では登録エラーとし、フリーテキストで表現する。
- **タイムゾーン**: 営業中判定は日本時間（Asia/Tokyo）固定（v1は東京のみ, 001/FR-022）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-101**: カフェは任意で 電話番号・予約情報（テキスト）・営業時間（フリーテキスト）・営業時間（曜日別の構造化）・公式リンク集（website/instagram/x/tabelog/googlemap/other）・犬向け設備（店内OK/テラスOK/大型犬OK/犬用メニュー、各 ✓/✕/不明）・犬設備の自由記述・運営転記メモ（本文＋出どころ＋確認日）・基本情報の確認日 を保持できなければならない (MUST)。
- **FR-102**: 構造化営業時間を持つカフェには、現在時刻（Asia/Tokyo）に基づく「営業中／営業時間外／本日定休」バッジを一覧・詳細に表示しなければならない (MUST)。構造化を持たないカフェにはバッジを表示してはならない (MUST NOT)（推測で「営業中」と示さない）。
- **FR-103**: 運営転記メモは出どころ（例: Instagram）と確認日を必須とし (MUST)、公式の一次情報そのものではないことが分かる形で表示する。確認日のないメモは配信してはならない (MUST NOT)。
- **FR-104**: 未登録の情報は非表示または「不明」として提示し、「なし/✕」と誤認させてはならない (MUST NOT)。
- **FR-105**: 追加情報はすべて既存のデータ契約（cafes.json format_version 1）への**後方互換の追加**として配信し、検証（時刻形式・URL形式・enum・確認日必須）を通過しないデータは配信してはならない (MUST NOT)。
- **FR-106**: 電話番号はタップで発信でき、公式リンクは外部で開けなければならない (MUST)。
- **FR-107**: カフェは任意で 地区名（sub_area, 例: 天王洲アイル）・店舗紹介（description）・犬のサイズ制限メモ（dog_size_limit）・定休日メモ（holiday_note）を保持し、詳細画面に表示できなければならない (MUST)。入力値 `unknown`/`不明` は「不明」として扱い、「なし/✕」と混同してはならない (MUST NOT)。Instagram のリンクは @ハンドル入力を URL に正規化して受け付ける。

### Key Entities（001 からの追加分）

- **公式リンク (CafeLink)**: 種別（website/instagram/x/tabelog/googlemap/other）とURL。
- **営業時間 (OpeningHours)**: 曜日→時間帯配列（open/close, "HH:mm"）。定休曜日は空配列。
- **犬向け設備 (DogAmenities)**: indoor/terrace/large_dogs/dog_menu（各 true/false/不明=null）。
- **運営転記メモ (OperatorNote)**: 本文・出どころ（instagram等）・確認日（必須）。

## Success Criteria *(mandatory)*

- **SC-101**: 情報が登録されたカフェについて、利用者は詳細画面から離脱せずに「営業中か・犬設備・予約方法・公式リンク」を確認できる（外部遷移は任意）。
- **SC-102**: 「営業中」バッジの判定が構造化データに対して100%正確（判定ロジックはユニットテストで担保）。
- **SC-103**: 未登録項目が「なし/✕」と表示されるケースが0件（不明は不明と表示）。
- **SC-104**: 確認日のない転記メモ・不正な時刻/URLを含むデータの配信が0件（エクスポート検証で拒否）。

## Assumptions

- 001 の構成B（Sheet → 検証済み cafes.json → バンドル＋静的URL）を前提とし、Sheet の列追加＋エクスポートスクリプト拡張で実現する。
- 営業中判定は端末内で行い、タイムゾーンは Asia/Tokyo 固定（v1）。
- アプリ内 Instagram 投稿埋め込み・写真表示は v1 スコープ外だったが、**v1.5（2026-07）で採用**: IG代表投稿の公式形式埋め込み（詳細画面のみ・読み込み失敗時は自動でOGP写真カードへフォールバック）＋OGP写真プレビュー（画像の転載・再ホストなし。端末が直接取得）＋タップでアプリ内ブラウザ＋一覧サムネイル＋リンク皆無店向けの地図検索リンクfallbackを追加。IGアカウント（プロフィール）の埋め込みは公式に存在しないため引き続き不採用。「営業中のみ」等のフィルタは引き続き v1 スコープ外（v2 候補）。
- 憲章の全原則（特に I: 出典・確認日・誤認させない／VI: 日本語ファースト）に従う。
