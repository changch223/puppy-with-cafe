# Quickstart / Validation Guide: Puppy With Cafe（構成B版）

フィーチャーがエンドツーエンドで動くことを検証する手順。**2026-07-05 に構成B（サーバーレス: Sheet → 検証済み cafes.json → バンドル＋静的URL配信）へ改訂**（research.md R11）。旧A案（Supabase）の手順は本書から削除し、SQL・コードは `supabase/` 等に保管。

## 前提
- macOS + Xcode（iOS 16+ シミュレータ/実機）
- Python 3（データエクスポート用・標準ライブラリのみ）
- （配信を有効にする場合）GitHub リポジトリ＋Pages ／（報告を有効にする場合）Google フォーム — 手順は `tools/README.md`

## セットアップ
1. `DokoWanCafe/DokoWanCafe.xcodeproj` を Xcode で開いて Run。**バンドル済みの `cafes.json`（架空サンプル7件）でそのまま動く**（オレンジの「サンプルデータ表示中」バナーが出る）。
2. データを変えたい場合: `tools/sheet_template/*.csv` を編集（または Google Sheet からダウンロード）→ `python3 tools/export_cafes.py`（実データなら `--sample` なし）→ 再ビルド。
3. 遠隔更新を試す場合: `AppConfig.defaultCafesDataURL`（または環境変数 `CAFES_DATA_URL`）に cafes.json のURLを設定。
4. 報告フォームを試す場合: `AppConfig.defaultReportFormTemplate`（または環境変数 `REPORT_FORM_URL`）にプリフィルURLを設定。

## 実行
- シミュレータの位置を東京（例: 東京駅 35.6812, 139.7671）に設定してアプリ起動。

## 検証シナリオ（ユーザーストーリー対応）

### US1（P1・MVP）現在地から発見
- 位置許可後、**5秒以内**に地図中心＝現在地、周辺の犬OKカフェがピン＋一覧（距離つき）で表示（SC-001/002）。
- 一覧は距離昇順。フィルタ「可のみ」で不可/未確認が消える。地図と一覧の内容が一致する。
- ピンが近接すると**クラスタ**（茶色の数字ピン）にまとまる。

### US2（P2）信頼情報
- 詳細で可否ステータス・（条件付きなら）条件・**出典と最終確認日**が表示される。
- 出典の無い「サンプル喫茶 神田」は**未確認**として区別表示。確認日が古い「サンプル・レトロ喫茶 銀座」に**古い可能性**の警告（FR-010）。

### US3（P2）誤り報告（構成B: フォーム）
- 詳細の「情報の誤りを報告」→ フォームURL 未設定なら**準備中**表示、設定済みなら店名入りで**フォームが開く**（サインイン不要, FR-028改訂）。
- 送信しても**表示は変わらない**。運営が Sheet を修正し `export_cafes.py` → 配信更新した時点で反映（FR-024）。差分は実行時表示と `data/CHANGELOG.md` で確認（FR-033）。

### US4（P3）矛盾・AI区別
- 「サンプル珈琲 日本橋」で**出典間の食い違い**バナー＋各出典の可否/確認日/由来、代表に「採用根拠」マーク（SC-007）。
- 「サンプルベーカリー 京橋」の出典に紫の **AI推測** ラベル（SC-004）。

### US5（P3）詳細＋経路
- 詳細の「経路案内（マップで開く）」→ 外部マップが当該カフェを目的地に開く。住所・出典リンクが見える。

### 横断（オフライン/プライバシー/a11y/エリア外）
- 機内モードでもバンドル/キャッシュデータで閲覧できる（FR-029）。遠隔取得成功後は「データ更新: ○月○日時点」バナー（FR-032）。
- 位置許可を拒否 → 「地域を選んで探す」で東京の主要エリアから検索できる（FR-017）。
- 位置を大阪に設定 → 「**この場所は現在サービス対象外です**」（0件と誤認させない, FR-022）。
- VoiceOver / Dynamic Type で主要フローが操作・可読（FR-031）。

## 自動テスト（憲章IV）
```bash
cd DokoWanCafe
xcodebuild test -project DokoWanCafe.xcodeproj -scheme DokoWanCafe \
  -destination 'platform=iOS Simulator,name=iPhone 17'
```
- 41テスト: 距離/フィルタ/名寄せ/矛盾解決(FR-013全分岐)/鮮度/エリア判定/**静的データ契約**（半径・可否・距離昇順・出典・矛盾、SC-005 相当）。

## 合格条件
- SC-001/002/003/004/005/007 が上記シナリオで確認できること。
- `export_cafes.py` が不正データ（条件なしの条件付き・確認日なしの確定情報・重複）を**エラーで拒否**すること（憲章 原則I）。
